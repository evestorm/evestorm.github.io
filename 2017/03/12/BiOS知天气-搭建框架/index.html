BiOS知天气-搭建框架

发表于 2017-03-12 | 分类于 iOS | | 阅读次数

前言

半年前，Udacity 与 滴滴 合作推出 BiOS课程 知天气项目，需要应用范例，我为此分别提供了 初级、中级 和 高级版本。由于当时使用的还是 Swift 2.3 ，在现在的 Xcode 环境下打开会报一堆错误信息。权衡后考虑使用 Swift3.0 再写一套出来。

需求

在高级版中，应用需求如下：

  • 通过 Table View 列出中国主要热门城市，用于选定某个城市查看天气；
  • 根据选择的城市，拉取天气数据。数据可从 free weather api 获取；
  • 图形显示天气数据显示，至少包括：当天及 未来3天 的天气状况（晴、多云……）和温度；
  • 下拉拖动手势自动刷新
  • 提供搜索框输入城市信息 （支持汉字及拼音）；
  • 自动定位并显示天气数据

​

准备

清楚需求后，就要开始搭建项目了。新建项目构建版本不提，在导入第三方上让我发愁了，太久没使用Swift编写项目，已经忘了该使用什么样的第三方了。为此我花了2个多小时在网上搜索浏览了一遍，最终敲定了以下几个为我所用：

网络部分
  • Alamofire 著名的 AFNetworking 网络基础库 Swift 语言版
  • HandyJson json解析库
下拉刷新
  • MJRefresh 用法简单的下拉刷新框架 （OC）
约束
  • SnapKit Masonry框架作者推出的 Swift 版布局框架

现阶段只用 CocoaPods 集成了上述几个第三方库，后续如果还有添加会在此更新。

搭建

本项目依然采用我熟悉的 MVVM 架构搭建，初步构想后我把该应用分为3大板块：

  • 主界面
  • 搜索页
  • 我的城市列表

这三大板块我认为已经能够满足基本的项目需求。

划分好项目层次结构后，就开始创建基类了（这次的天气应用由于不需要用到底部切换，就省去了UITabBarController）：

  • NavigationViewController
    • 导航栏标题颜色设置
    • 导航栏透明设置
  • BaseViewController
    • 界面的背景色
    • 导航状态栏颜色切换
    • 刷新控件集成

编写完基类后，又开始基于 Alamofire 封装网络请求工具类：EVENetworkManager

有关定位我也对其进行了封装：EVELocationManager

做完这么多后，一个基本的项目框架就搭建完成了。

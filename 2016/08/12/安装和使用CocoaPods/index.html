安装和使用CocoaPods

发表于 2016-08-12 | 分类于 iOS | | 阅读次数

此安装只针对Mac 10.11以上版本

CocoaPods简介

CocoaPods是一个负责管理iOS项目中第三方开源库的工具。CocoaPods的项目源码在Github上管理。该项目开始于2011年8月12日，在这两年多的时间里，它持续保持活跃更新。开发iOS项目不可避免地要使用第三方开源库，CocoaPods的出现使得我们可以节省设置和更新第三方开源库的时间

更新gem

  • sudo gem update –system

    更新ruby的软件源

  • gem sources –remove https://rubygems.org/

  • gem sources -a https://ruby.taobao.org/

  • gem sources -l

安装CocoaPods

  • sudo gem install -n /usr/local/bin cocoapods

过一会会报错：

activesupport requires Ruby version >= 2.2.2.

提示是 ruby的版本需要大于等于2.2.2，所以先要更新ruby，这里使用RVM对Ruby进行升级，升级过程如下：

  1. 安装 RVM
  • curl -L get.rvm.io | bash -s stable
  1. 等待一段时间后就可以成功安装好 RVM。

  • source ~/.bashrc

  • source ~/.bash_profile

  1. 测试是否安装正常
  • rvm -v
  1. 用RVM升级Ruby

查看当前ruby版本

  • ruby -v

列出已知的ruby版本

  • rvm list known

安装ruby 2.3.0

  • rvm install 2.3.0

    安装过程中可能会停下来，提示你请选择安装路径，此时只用回车就行了

  1. 安装完成之后ruby -v查看是否安装成功。

安装pod

  • pod setup

这个操作会耗费大量时间，请耐心等待（我安装完成大概用了40分钟左右…）

另外在等待过程中，你还可能经历报错，针对报错的小伙伴，我已经将报错后的解决办法整理了出来：

第一次报错
Setting up CocoaPods master repo
[!] /usr/bin/git clone https://github.com/CocoaPods/Specs.git master

Cloning into ‘master’…
error: RPC failed; result=56, HTTP code = 200
fatal: The remote end hung up unexpectedly
fatal: early EOF
fatal: index-pack failed

  • sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer
    Password:

第二次报错

Cloning into ‘master’…
fatal: unable to access > ‘https://github.com/CocoaPods/Specs.git/‘: Server aborted the SSL handshake

  • sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer
    Password:

第三次就不报错了，坐等一段时间提示成功就OK了

CocoaPods安装东西的时候它要找到Xcode的Developer文件夹, 如果找不到会报如下错误

xcrun: error: invalid active developer path (/Library/Developer/CommandLineTools), missing xcrun at: /Library/Developer/CommandLineTools/usr/bin/xcrun

最后的解决方法：

/Applications/Xcode-beta.app/Contents/Developer这个路径 你可以打开Xcode显示包内容，找到那个文件夹拖到终端里面比较不容易错。

使用cocoapods之后的注意事项

  1. 利用CocoaPods管理类库后, 以后打开项目就要打开工作空间了：xxxx.xcworkspace , 而不是打开
    之前的.xcodeproj文件

  2. 每次更改了Podfile文件,你需要重新执行一次更新命令。

    pod update

  3. CocoaPods在执行pod
    install和pod update时,会默认先更新一次CocoPods的spec仓库索引。使用–no-repo-update参数可以禁止其做索引更新操作

pod install
–no-repo-update
